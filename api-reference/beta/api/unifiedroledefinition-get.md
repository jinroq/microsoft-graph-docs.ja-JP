# <a name="get-unifiedroledefinition"></a>UnifiedRoleDefinition を取得する

[!INCLUDE [beta-disclaimer](../../includes/beta-disclaimer.md)]

[UnifiedRoleDefinition](../resources/unifiedRoleDefinition.md)オブジェクトのプロパティとリレーションシップを取得します。 サポートされている唯一の RBAC アプリケーションは、現在 "ディレクトリ" です。

## <a name="permissions"></a>アクセス許可

この API を呼び出すには、次のいずれかのアクセス許可が必要です。アクセス許可の選択方法などの詳細については、「[アクセス許可](/graph/permissions-reference)」を参照してください。

|アクセス許可の種類      | アクセス許可 (特権の小さいものから大きいものへ)              |
|:--------------------|:---------------------------------------------------------|
|委任 (職場または学校のアカウント) | RoleManagement、RoleManagement、Directory.accessasuser.all、および all のいずれかを取得します。すべてのディレクトリについては。    |
|委任 (個人用 Microsoft アカウント) | サポートされていません。    |
|アプリケーション | RoleManagement、RoleManagement、および All のいずれかのディレクトリを参照しています。 |

## <a name="http-request"></a>HTTP 要求

<!-- { "blockType": "ignored" } -->

```http
GET /roleManagement/{rbacApplication}/roleDefinitions/{id}
```

## <a name="optional-query-parameters"></a>オプションのクエリ パラメーター

このメソッドは、応答をカスタマイズするための OData クエリ パラメーターをサポートします。 一般的な情報については、「 [OData クエリパラメーター](/graph/query-parameters)」を参照してください。

## <a name="request-headers"></a>要求ヘッダー

| 名前      |説明|
|:----------|:----------|
| Authorization | ベアラー {トークン} |

## <a name="request-body"></a>要求本文

このメソッドには、要求本文を指定しません。

## <a name="response"></a>応答

成功した場合、このメソッド`200 OK`は応答コードと、応答本文で要求された[unifiedRoleDefinition](../resources/unifiedroledefinition.md)オブジェクトを返します。
